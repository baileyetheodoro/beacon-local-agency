# Beacon Local Reusable Website Templates

These templates are based on the Beacon Local service packages:

- **Launch Site**: Up to 3 pages, simple professional presence, contact form, basic SEO.
- **Professional Site**: Up to 5 pages, stronger credibility, gallery/portfolio, FAQ, more organized service content.
- **Growth Site**: Up to 10 pages, dedicated service pages, blog setup, analytics-ready structure, deeper navigation.

## Main Multi-Page Templates

When starting a client site, pick the closest package template first:

- Use `launch/` for new businesses, side hustles, solo professionals, and simple service businesses.
- Use `professional/` for established local businesses with several services, reviews, project examples, or a portfolio.
- Use `growth/` for businesses with multiple service categories, locations, education/blog needs, or future expansion plans.

Each folder is a complete starter website:

- `launch/`: `index.html`, `services.html`, `contact.html`
- `professional/`: `index.html`, `services.html`, `about.html`, `portfolio.html`, `contact.html`
- `growth/`: `index.html`, `about.html`, `services.html`, `service-one.html`, `service-two.html`, `service-three.html`, `faq.html`, `blog.html`, `contact.html`

Each folder also includes:

- `styles.css`: shared visual styling and responsive layout rules for that package.
- `menu.js`: mobile menu behavior.

Each page uses placeholder tokens such as `{{BUSINESS_NAME}}`, `{{PRIMARY_SERVICE}}`, and `{{CITY_OR_AREA}}`. Replace those first, then customize:

1. Brand colors and fonts.
2. Hero message and call to action.
3. Services and proof points.
4. Photos, gallery, or portfolio examples.
5. Contact details and form destination.
6. SEO title, description, and local keywords.

The original one-page scroll versions are still included as reference files:

Use those only when you want a single-page demo. For client projects, the folder-based templates are the better starting point because they match Beacon Local's package scopes.

## Reuse Notes

These templates should reduce design and build time because the page strategy, section order, responsive layout, and conversion flow are already decided. They do not remove the need for customization. Each business still needs its own messaging, service hierarchy, images, local SEO language, and trust signals.

Best workflow: choose a template, gather client details, customize the content and visual direction, then polish the site for that exact business.
